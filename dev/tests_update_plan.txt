2. test/yarpecule/graph/test_smiles.py

What changed
- `smiles2adjmat()` still returns 3 values, but `atom_info` is now a dict keyed by atom index.
- Old tests assume list rows like `label[0]`.

Global replacement pattern
- Replace:
  - `elements = [label[0] for label in atom_info]`
- With:
  - `elements = [atom_info[i]["element"].capitalize() for i in atom_info]`

Why capitalize
- Production parser now stores lowercase element labels in metadata.
- Existing test expectations use uppercase element symbols.

Main edit sites

3. test/yarpecule/test_yarpecule.py

What changed
- `get_smiles()` now uses preserved `_atom_info["atom_map"]` values instead of RDKit atom indices.
- Missing atom maps are generated at yarpecule creation.
- As a result, mapped-smiles strings can change while canonical smiles should remain stable.

Tests to rebaseline
- `TestSMILESFromYarpecule.test_haa_mapping`
- `TestSMILESFromYarpecule.test_aro_mapping`
- `TestSMILESFromYarpecule.test_rad_mapping`

What to keep
- Keep the `canon_smi` assertions if they still match runtime output.

What to update
- Rebaseline the `map_smi` expected strings to the new preserved-map behavior.

Suggested helper workflow
- For each of the three tests:
  1. create the yarpecule with `canon=False`
  2. call `get_smiles()`
  3. print `mol.map_smi`
  4. paste the new exact string into the assertion

Useful extra assertions
- `assert [mol._atom_info[i]["atom_map"] for i in mol._atom_info] == [...]`
