Implementations And Questions Log
================================

Status Summary
--------------
- Planning docs updated to use dict-based `_atom_info` with keyed fields.
- RDKit generation path updated in plan to be inherently stereo-free (`isomericSmiles=False`) rather than post-strip.
- Fallback policy set to warning+fallback by default.
- Isotope policy set to integer override in a single `mass` field.
- Map-aware XYZ/MOL export included in plan.

Implementation Decisions (Current)
----------------------------------
1. `_atom_info` schema
   - dictionary keyed by local atom index
   - key fields: `atom_index`, `atom_map`, `element` (uppercase), `formal_charge`, `mass`, `stereo`, `aromatic_input`

2. Mapping logic
   - preserve all user-provided map values
   - duplicate mapped input -> hard error
   - fill missing only
   - fill order: heavy atoms then hydrogens
   - start at 0 and skip used values

3. Stereo policy
   - store `@`, `@@`, `/`, `\\` in `_atom_info`
   - never emit stereo in either YARP or RDKit output strings
   - RDKit mode uses generation flags to keep output stereo-free intrinsically

4. Aromatic handling
   - non-kekulization-first approach
   - deterministic aromatic assignment attempt
   - warning+fallback default if assignment fails

5. Canonicalization
   - target: YARP-independent canonicalization in dedicated writer path
   - keep RDKit mode as fallback/compatibility path

6. Export behavior
   - MOL: map-aware output via atom mapping field
   - XYZ: keep standard XYZ body for compatibility; add map metadata in comment line (`maps=...`)

Slide-Transfer Section
----------------------
Slide 1: Problem and Constraints
- Need deterministic, reviewable SMILES generation with preserved atom mapping for ML workflows.
- Current parser/writer path loses or rewrites mapping in some workflows.
- Requirements: dict-based atom metadata, stereo retained as metadata only, canonical uppercase explicit-bond output.

Slide 2: Proposed Architecture
- Parser (`smiles.py`) keeps graph + `_atom_info` metadata (maps, stereo, mass, aromatic hints).
- New writer module (`graph/smiles_yarp.py`) performs canonical ranking + traversal + emission.
- YARP path is primary; RDKit path remains fallback/compatibility.
- Join/separate carry `_atom_info` with collision-safe reindexing and no separation renumbering.

Slide 3: Risk Control and Validation
- Duplicate map input errors early.
- Aromatic resolution uses warning+fallback default.
- `%10+` ring tokens supported.
- Unit tests mirror existing class-based style and fixtures.
- Map-aware exports: MOL standard field, XYZ comment metadata for Avogadro/VMD readability.

Open Questions (for team discussion)
