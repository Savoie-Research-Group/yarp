Thought for 1m 8s

Here is the design I’d use for a primary-path custom canonical writer built around your parser’s atom dict, with toolkit use limited to an optional oracle/debug path.

The main design choice is to treat your output as a project-defined canonical SMILES dialect: uppercase atoms, all bonds explicit, no emitted stereo, and optional atom maps. That is slightly different from the usual aromatic-output preference in OpenSMILES, but it is completely reasonable because OpenSMILES standardizes syntax more than it standardizes one universal canonicalization algorithm, and the literature is explicit that there is no single cross-tool canonical SMILES standard in common use. Daylight’s historical approach is a two-stage process—canonical labeling, then canonical generation—which is the right model to copy here.

Recommended writer contract

Your writer should guarantee these invariants:

Atom maps

preserve every user-supplied atom_map exactly

reject duplicate mapped input before any canonicalization

fill only missing map IDs

fill order: canonical heavy atoms first, then canonical hydrogens

canonical ranking must ignore atom_map

Stereo

keep stereo["atom"] and stereo["bonds"] in metadata only

never emit @, @@, /, or \

Output forms

canonical_smiles: uppercase, explicit-bond, stereo-free

canonical_mapped_smiles: same traversal/order, but with map IDs attached

Ring closures

support 0–9 directly

support >=10 with %10, %11, etc., which is standard SMILES syntax for higher-numbered closures.

Canonicalization algorithm
1) Validate first

Do these checks before anything else:

every bond references valid atom indices

graph is connected if you only support single-fragment output, or fragments are sorted canonically if you support . disconnections

duplicate atom_map values raise immediately

stereo fields are preserved but ignored by ranking/emission

aromatic flags are not trusted for output; they are only an input hint

2) Aromatic localization before ranking

Because your output is uppercase explicit-bond SMILES, lowercase aromatic input should be converted into a localized single/double-bond form before canonical ranking. That is also the cleanest way to avoid toolkit-dependent aromatic output. Daylight explicitly allows aromatic structures to be supplied either directly or in Kekulé form, and OpenSMILES notes that aromatic output is usually preferred; your writer is intentionally choosing the opposite for determinism and reviewability.

Recommended deterministic strategy:

collect each connected subgraph induced by aromatic_input=True

for each component, build a constrained bond-order assignment problem:

candidate aromatic edges may become single or double

atom valence must remain legal

formal charge must be preserved

explicit hydrogens must be preserved

solve by:

first trying a deterministic maximum-matching / perfect-matching style assignment on the aromatic component

if multiple valid assignments exist, choose the one with the lexicographically smallest bond-order signature under provisional atom ranks

if no valid assignment exists:

warn

optionally consult a toolkit oracle in debug mode only

if still unresolved, either:

raise in strict=True, or

fall back in warn mode to a conservative all-single assignment only if valence remains safe

That “warn + deterministic fallback” is safer than “always kekulize with a toolkit,” because toolkit aromaticity models differ. RDKit explicitly documents multiple aromaticity models and also documents that kekulized output is not canonical and may fail if the molecule cannot be kekulized.

3) Atom invariants for ranking

Use atom invariants that are independent of input order and map IDs:

element

formal charge

mass

explicit hydrogen count if your parser stores it elsewhere

degree

valence / bond-order sum

ring membership

localized bond-order multiset

aromatic provenance only as a late tiebreaker, not as an emitted property

Do not include:

atom_map

atom stereo

bond stereo

The reason to exclude maps is both conceptual and practical: mapped and unmapped variants of the same graph should canonicalize to the same plain SMILES. RDKit even exposes an ignoreAtomMapNumbers switch for this reason in canonicalization, which makes it a good oracle behavior to match, though not your primary implementation.

4) Rank refinement

Use a Morgan/WL-style iterative refinement:

initialize each atom with its invariant tuple

iteratively replace each label with:

current label

sorted multiset of (bond_order, neighbor_label) pairs

compress labels to integers

repeat until stable

break any remaining symmetries with deterministic secondary rules:

deeper neighborhood signatures

smallest canonical branch code

original atom index only as final last-resort deterministic fallback

This is not identical to published Daylight CANGEN internals, but it follows the same two-stage spirit: canonical labeling first, string generation second. The original unique-SMILES paper describes canonical labeling followed by generation from the lowest-ranked atom.

5) Canonical traversal and emission

After final ranks:

choose the lowest-ranked atom as root

build a canonical DFS tree

at each atom, sort neighbors by:

bond order priority

neighbor rank

subtree code / branch code

first child stays in the main chain

later children become branches in (...)

non-tree edges become ring closures

For emission:

always print explicit bonds: -, =, #

always print uppercase elements

never print stereo

write bracket atoms when needed:

non-organic subset

charged atoms

isotopes / mass

mapped atoms in mapped mode

explicit hydrogens if your writer supports them

ring numbers:

1–9 as single digits

10+ as %10, %11, etc.

Reference implementation sketch


from collections import defaultdict
from typing import Dict, List, Tuple, Optional

AtomTable = Dict[int, dict]
BondTable = List[dict]


class CanonSmilesError(Exception):
    pass


class DuplicateAtomMapError(CanonSmilesError):
    pass


def validate_atom_maps(atoms: AtomTable) -> None:
    seen = {}
    for idx, atom in atoms.items():
        amap = atom.get("atom_map")
        if amap is None:
            continue
        if amap in seen:
            raise DuplicateAtomMapError(
                f"duplicate atom_map={amap} on atoms {seen[amap]} and {idx}"
            )
        seen[amap] = idx


def build_adjacency(atoms: AtomTable, bonds: BondTable) -> dict[int, list[int]]:
    adj = {idx: [] for idx in atoms}
    for bidx, bond in enumerate(bonds):
        a1, a2 = bond["a1"], bond["a2"]
        adj[a1].append(bidx)
        adj[a2].append(bidx)
    return adj


def atom_invariant(idx: int, atoms: AtomTable, bonds: BondTable, adj: dict[int, list[int]]) -> tuple:
    atom = atoms[idx]
    bond_orders = []
    for bidx in adj[idx]:
        bond = bonds[bidx]
        bond_orders.append(bond["order"])
    bond_orders.sort()
    return (
        atom["element"],
        atom.get("formal_charge", 0),
        atom.get("mass"),
        len(adj[idx]),
        tuple(bond_orders),
    )


def refine_canonical_ranks(atoms: AtomTable, bonds: BondTable) -> dict[int, int]:
    adj = build_adjacency(atoms, bonds)
    labels = {idx: atom_invariant(idx, atoms, bonds, adj) for idx in atoms}

    for _ in range(12):
        new_labels = {}
        for idx in atoms:
            nbr_sig = []
            for bidx in adj[idx]:
                bond = bonds[bidx]
                nbr = bond["a2"] if bond["a1"] == idx else bond["a1"]
                nbr_sig.append((bond["order"], labels[nbr]))
            nbr_sig.sort()
            new_labels[idx] = (labels[idx], tuple(nbr_sig))

        uniq = {lab: i for i, lab in enumerate(sorted(set(new_labels.values())))}
        compressed = {idx: uniq[lab] for idx, lab in new_labels.items()}
        if labels == compressed:
            break
        labels = compressed

    ordered = sorted(
        atoms.keys(),
        key=lambda i: (labels[i], atom_invariant(i, atoms, bonds, adj), i)
    )
    return {idx: r for r, idx in enumerate(ordered)}


def choose_root(ranks: dict[int, int]) -> int:
    return min(ranks, key=lambda i: ranks[i])


def fill_missing_atom_maps(atoms: AtomTable, ranks: dict[int, int]) -> None:
    validate_atom_maps(atoms)
    used = {a["atom_map"] for a in atoms.values() if a.get("atom_map") is not None}
    next_id = 0

    def alloc() -> int:
        nonlocal next_id
        while next_id in used:
            next_id += 1
        out = next_id
        used.add(out)
        next_id += 1
        return out

    heavy = sorted(
        [i for i, a in atoms.items() if a.get("atom_map") is None and a["element"] != "H"],
        key=lambda i: ranks[i],
    )
    hydrogens = sorted(
        [i for i, a in atoms.items() if a.get("atom_map") is None and a["element"] == "H"],
        key=lambda i: ranks[i],
    )

    for idx in heavy + hydrogens:
        atoms[idx]["atom_map"] = alloc()


def format_atom(atom: dict, mapped: bool) -> str:
    el = atom["element"].upper()
    mass = atom.get("mass")
    charge = atom.get("formal_charge", 0)
    amap = atom.get("atom_map")

    simple_subset = el in {"B","C","N","O","P","S","F","CL","BR","I"} and not mass and charge == 0 and not mapped
    if simple_subset:
        return el

    parts = ["["]
    if mass is not None:
        parts.append(str(mass))
    parts.append(el)
    if charge == 1:
        parts.append("+")
    elif charge > 1:
        parts.append(f"+{charge}")
    elif charge == -1:
        parts.append("-")
    elif charge < -1:
        parts.append(str(charge))
    if mapped:
        parts.append(f":{amap}")
    parts.append("]")
    return "".join(parts)


def format_ring_number(n: int) -> str:
    return str(n) if n < 10 else f"%{n}"

Robust test matrix

I’d structure tests by behavior rather than by molecule family.

1) Atom-map tests

all mapped, unique → preserved exactly

duplicate mapped input → raises

partially mapped → only missing are filled

maps starting at nonzero → preserved, missing start at 0 and skip used IDs

heavy+H mixture → heavy atoms filled before hydrogens

7) Regression/oracle tests

Recommended tie-break rules

When rank refinement leaves ties, use a stable cascade:

smaller invariant tuple

lexicographically smaller sorted neighbor signature

lexicographically smaller recursively generated branch code

lower atom index as final deterministic fallback only

That last rule is not chemically meaningful, but it is acceptable as a final total-order guarantee for your own writer.

Bottom line

I would implement this as:

validate graph + reject duplicate maps

localize lowercase aromatic input into deterministic explicit bond orders

compute map-agnostic canonical atom ranks

fill missing maps only, using canonical heavy-then-H order

emit one stereo-free uppercase explicit-bond SMILES

emit one mapped variant using the same canonical traversal

That gives you a writer that is deterministic, reviewable, independent of toolkit canonicalization in the primary path, and explicit about how aromatic ambiguity is handled.
